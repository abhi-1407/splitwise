Splitwise

A Splitwise clone built to practice backend engineering, low-level design (LLD), object-oriented design (OOD), design patterns, Spring Boot, and relational database design.

The project began as a pure Java LLD implementation and has since been migrated to a Spring Boot application backed by PostgreSQL using Spring Data JPA.

Database Design

The application persists the following entities:

* User
* Group
* Expense
* Split

Entity relationships are managed using Hibernate annotations such as:

* @OneToMany
* @ManyToOne
* @ManyToMany
* @JoinColumn
* @JoinTable
* Cascade Operations
* Lazy Loading

UUIDs are generated automatically using Hibernate.

Design Decisions

Dynamic Balance Computation

Balances are not stored in the database.

Instead, they are computed dynamically from persisted Expense and Split records whenever requested.

This approach:

* Eliminates redundant state
* Avoids synchronization issues
* Maintains a single source of truth
* Keeps the database normalized

Transaction Management

Expense creation is performed within a transaction so that an expense and all associated splits are persisted atomically.

UUID Primary Keys

All entities use UUIDs generated by Hibernate, avoiding manual identifier generation.

⸻

Debt Simplification

The application computes each user’s net balance and minimizes the number of required transactions using a greedy algorithm backed by Priority Queues.

Example:

Original Debts

* A owes B ₹100
* B owes C ₹100

Simplified Result

* A owes C ₹100

This reduces unnecessary intermediate transactions.
